## Dashboardtest Codebase Findings (2026-02-20)

### Snapshot
- Stack: Next.js `16.1.6` + React `19`, App Router under `src/app`.
- Auth: NextAuth v5 beta with Google provider (`src/auth.ts`), JWT sessions.
- Tooling: Biome (`npm run check|lint|format`), Jest, Playwright.
- Runtime: Node `>=20.9.0` (`package.json`, `.nvmrc`).
- Important build setting: `next.config.mjs` has `typescript.ignoreBuildErrors = true`.

### Main App Areas
- Pages: `/`, `/form`, `/clientes`, `/recorridos`, `/inventario`, `/inventario-carro`, `/navegar`, `/transacciones`, `/buzon`, `/inspector-periodos`, `/admin`, auth pages.
- `src/app/layout.tsx` wraps app with `AuthProvider`, `MotionProvider`, registers SW (`/service-worker.js`), and includes `DevAgentation` in development.
- Navigation lives in `src/components/AppHeader.tsx`; `/transacciones` menu item is shown only for master accounts.

### Auth and Roles
- Middleware (`src/middleware.ts`) protects almost all routes and redirects unauthenticated users to `/auth/signin`.
- Allowed sign-in emails are enforced in `src/auth.ts` callback.
- Role helpers and vendor mappings are in `src/utils/auth.ts`:
  - `MASTER_ACCOUNTS`
  - `isInventarioCarroAdmin`
  - `EMAIL_TO_VENDOR_LABELS`
  - vendor alias/normalization helpers

### Data and Integrations
- Primary datastore is Google Sheets (many routes/components use spreadsheet + tab ranges directly).
- API route handlers in `src/app/api/**` cover:
  - form submissions (`submit-form`)
  - clients/analytics (`clientes`)
  - routes scheduling/updates (`recorridos`, `recorridos/update`, `recorridos/reschedule`)
  - transactions feed (`transacciones`)
  - inventory (`inventario-carros`, `inventory/add-entrada`)
  - inbox/feed review (`buzon-unseen`, `feed-reviews`)
  - auth handlers (`auth/[...nextauth]`, `auth/token`)
  - photo upload (`upload-photo`) using Cloudflare R2 via S3 SDK
- Several client pages also fetch Google Sheets API directly using `NEXT_PUBLIC_GOOGLE_API_KEY`.

### Offline/PWA
- Service worker: `public/service-worker.js` with background sync (`order-submission-sync`).
- Offline queue: `src/utils/submissionQueue.ts` + `src/hooks/useSubmissionQueue.ts`.
- Offline photo cache: `src/utils/photoStore.ts` (IndexedDB).

### Test Coverage Surface
- Unit tests focus on utilities (`src/__tests__/utils/dateUtils.test.ts`, `src/__tests__/utils/transacciones.test.ts`).
- E2E coverage includes queue behavior (`tests/e2e/queue-fixes.spec.ts`).

### Codebase Hotspots
- Large files to edit carefully:
  - `src/app/clientes/page.tsx` (~4039 lines)
  - `src/app/inspector-periodos/page.tsx` (~3013 lines)
  - `src/app/form/page.tsx` (~2319 lines)
  - `src/app/recorridos/page.tsx` (~1787 lines)
  - `src/app/api/clientes/route.ts` (~1597 lines)

### High-Priority Findings
- Hardcoded Google service account credentials are present in repository code (`src/utils/googleAuth.ts`, `src/app/api/clientes/route.ts`). Move to environment variables/secrets manager.
- `/api/auth/token` returns Google access tokens without explicit auth guard and is excluded by middleware matcher (`src/app/api/auth/token/route.ts`, `src/middleware.ts`). Treat as sensitive and protect/remove.
- Production build currently allows TypeScript errors (`next.config.mjs`), so CI/runtime regressions can slip through.
- Multiple API routes and utilities log detailed operational/user context; review and reduce sensitive logging for production.

### Working Conventions For Agents
- Keep the Next.js docs index block above intact; append project-specific notes below it.
- Prefer `@/` import alias (`tsconfig.json` paths).
- Use Biome commands for lint/format; this project does not use ESLint as the primary linter.
- Preserve period/week logic in `src/utils/dateUtils.ts` (periods start at `2024-10-05`, period numbering starts at `11`).
- Treat Google Sheets column mapping in `src/app/api/submit-form/route.ts` and `src/utils/transacciones.ts` as a strict contract before modifying.
